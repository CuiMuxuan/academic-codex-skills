# AI Image Generation Gate

Use this reference before OpenAI image generation or any AI-generated bitmap visual is used for an academic figure or supporting visual asset.

## Required Gate

Before generating:

1. Explain why SVG, Draw.io, Matplotlib, or native charting is insufficient.
2. Ask the user for explicit confirmation.
3. Confirm that the current environment can access the OpenAI image generation API or native image-generation tool.
4. Clarify output requirements: style, dimensions, transparency, labels, and whether text should be added afterward as vector text.
5. Record that the image is AI-generated if the target publication, school, or funder requires disclosure.

Do not call image generation silently.

## Appropriate Uses

Prefer AI generation for:

- photorealistic or illustrative assets;
- visual metaphors where exact geometry is not the scholarly claim;
- texture, scene, or cutout assets used as supporting visuals;
- base images that will later receive factual labels in an editable vector layer.

Avoid AI generation for:

- factual mechanisms with exact labels;
- quantitative results;
- code-backed architecture;
- diagrams where invented geometry would mislead the reader;
- any image that would imply experimental evidence not provided by the user.

## Academic Handling

- Generate base images without text when possible.
- Add factual labels, arrows, scale bars, and captions afterward in SVG, Draw.io, or another editable vector layer.
- If the target venue requires disclosure, include an explicit note in the project report or manuscript support materials.
- If the figure cannot be made faithful from the evidence, stop at a storyboard or request user-provided assets.
