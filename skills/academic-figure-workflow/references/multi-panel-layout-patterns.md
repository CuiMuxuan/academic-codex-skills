# Multi-Panel Layout Patterns

Use this reference for multi-panel, Nature-style, high-impact-journal, thesis chapter summary, or mixed-modality figures.

## Figure Contract First

Before layout, create:

```text
Main claim:
Reading order:
Hero panel:
Support panels:
Shared legend/key:
Panels to merge/drop:
Target width:
Final-size readability risk:
```

If a panel does not add unique evidence, merge or drop it.

## Standard Patterns

| Pattern | Use when | Layout rule |
|---|---|---|
| Quantitative grid | several comparable metrics or datasets | consistent axes, shared legend, aligned baselines, minimal repeated labels |
| Schematic-led composite | method/framework plus supporting results | give the schematic the largest top or left region; results validate it |
| Image plate plus quantification | images demonstrate phenotype/state and plots quantify it | keep image tiles aligned with matching quant panels and include scale bars |
| Asymmetric hero figure | one panel carries the core visual argument | let the hero panel span multiple grid cells; supporting panels stay compact |
| Clinical or longitudinal triptych | repeated outcomes over time or cohorts | keep columns aligned across rows and use a shared legend strip |
| Architecture plus workflow | system structure and runtime process | separate static layers from temporal flow; do not mix both in one tangled graph |

## Panel Ordering

- Use lowercase alphabetical panel labels whenever possible.
- Reading order should follow left-to-right, top-to-bottom unless the figure explicitly uses a different path.
- Keep labels near panel top-left, outside data when possible.
- Do not use boxed letters or decorative tag shapes.
- If a panel spans multiple cells, place its letter at the visual start of that panel.

## Legend Economy

- Use one shared legend for repeated categories.
- Use direct labels for fixed spatial regions, channels, or line identities.
- Put a legend-only area at the end or top of a row when it reduces clutter.
- Remove repeated legend titles when the axis title or caption already gives the meaning.

## Space Use

- Give panels enough space according to information density, not equality for its own sake.
- Use tight but readable gutters.
- Leave extra room for long tick labels, diagonal labels, colour bars, or scale bars.
- Avoid empty whitespace caused by trying to force all panels into identical rectangles.

## Revision Triggers

Revise the layout if:

- the main claim is not visible within five seconds;
- a reader must jump between distant legends and marks repeatedly;
- two panels answer the same question;
- any axis, legend, or label becomes unreadable at target width;
- an image panel is larger than the evidence it contributes;
- arrow directions or panel relationships require verbal explanation to decode.
