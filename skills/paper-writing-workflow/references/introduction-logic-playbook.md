# Introduction Logic Playbook

Use this reference for Introduction sections, thesis opening chapters, background/problem sections, and manuscript sections that must explain why the work exists.

## Required Logic Chain

An Introduction must connect these moves:

1. Research theme: define the broad field and the specific focus.
2. Problem layers: explain the wider problem, then narrow to the specific unresolved problem.
3. Importance: show why the specific problem matters for the target field or application.
4. Current work: summarize what prior work has achieved and where the remaining gap is, using evidence.
5. Purpose / 目的: state what this paper or thesis aims to do.
6. Contribution or work scope: state the completed work, analysis, system, experiment, or argument that addresses the purpose.

Do not start with a long background inventory unless it directly helps the reader reach the paper's purpose.

## Paragraph Roles

Typical Introduction paragraph jobs:

- field context;
- specific problem definition;
- prior-work synthesis;
- gap narrowing;
- purpose and contribution;
- manuscript organization when required.

Each paragraph should advance the reader from background toward the paper's purpose.

## Transition Requirements

- Use transition sentences when moving from broad field to specific problem.
- Make cause, contrast, and narrowing logic explicit.
- Avoid abrupt jumps from literature summary to "therefore this paper" without showing the gap.
- Avoid generic purpose statements that could fit any topic.

## Evidence Discipline

Evidence is required for:

- field-wide trends;
- claims about unresolved problems;
- statements that a method is widely used or insufficient;
- comparisons among prior approaches;
- claims that a problem affects practice, safety, cost, efficiency, or policy.

If evidence is missing, use `LIT_GAP` and route the gap to verification.
